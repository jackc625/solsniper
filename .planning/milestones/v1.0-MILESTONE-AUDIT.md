### Remaining Integration Gap

**INT-01: configSignal initialization timing** (minor, frontend-only)

`configSignal` in `dashboard/src/store/config.ts` starts as `{}` and is only populated when the Settings component mounts or after `saveConfig()`. The DRY RUN banner in `App.tsx` and Sidebar dry-run indicator read `configSignal.value?.dryRun` — invisible on fresh page load until user visits Settings tab.

- **Affects:** DRY-07 display only (banner visibility on initial load)
- **Does NOT affect:** Bot trading behavior, backend hot-reload, any other requirement
- **Fix:** Call `fetchConfig()` in App's mount useEffect alongside `connectFeed()`
