3. Create `eslint-rules/no-sql-template-literals.js` (per D-03):
   - Export a rule with `meta.type: 'problem'` and `meta.messages.noTemplateLiteral`
   - In `create(context)`, define `SQL_METHODS = new Set(['prepare', 'run', 'exec', 'all', 'get'])`
   - Return a `CallExpression` visitor that checks:
     - `node.callee.type === 'MemberExpression'`
     - `node.callee.property.type === 'Identifier'`
     - `SQL_METHODS.has(node.callee.property.name)`
     - For each argument: if `arg.type === 'TemplateLiteral' && arg.expressions.length > 0`, report
   - Message: `'Template literal in {{method}}() call -- use parameterized queries with ? or @named placeholders'`
   - NOTE: `arg.expressions.length > 0` is critical -- backtick strings with NO interpolation (like line 100 of trade-store.ts where `NON_TERMINAL_STATES.map(() => '?')` is passed as a spread arg, not inside the template) are SAFE. The template literal on line 100 does have an expression (`${NON_TERMINAL_STATES.map(() => '?').join(',')}`), but this generates only `?` placeholders from a hardcoded array. This is a known false positive that should be suppressed with an `// eslint-disable-next-line` comment in trade-store.ts.

4. Create `eslint-rules/no-api-key-in-url.js` (per D-06):
   - Export a rule with `meta.type: 'problem'` and `meta.messages.apiKeyInUrl`
   - Define `PATTERN = /[?&](api[-_]?key)=/i`
   - Return visitors for both `Literal` (check `typeof node.value === 'string'`) and `TemplateLiteral` (check each `quasi.value.raw`)
   - Message: `'API key pattern "{{match}}" found in string -- pass keys via headers instead'`

5. Create `eslint.config.js` (ESLint 10 flat config):
   ```javascript
   import tseslint from 'typescript-eslint';
   import noSqlTemplateLiterals from './eslint-rules/no-sql-template-literals.js';
   import noApiKeyInUrl from './eslint-rules/no-api-key-in-url.js';

   export default [
     {
       files: ['src/**/*.ts'],
       languageOptions: {
         parser: tseslint.parser,
         parserOptions: {
           ecmaVersion: 'latest',
           sourceType: 'module',
         },
       },
       plugins: {
         'security': {
           rules: {
             'no-sql-template-literals': noSqlTemplateLiterals,
             'no-api-key-in-url': noApiKeyInUrl,
           },
         },
       },
       rules: {
         'security/no-sql-template-literals': 'error',
         'security/no-api-key-in-url': 'error',
       },
     },
   ];
   ```

6. Add an `// eslint-disable-next-line security/no-sql-template-literals` comment above line 100 of `src/persistence/trade-store.ts` (the safe `NON_TERMINAL_STATES.map(() => '?')` template literal).
