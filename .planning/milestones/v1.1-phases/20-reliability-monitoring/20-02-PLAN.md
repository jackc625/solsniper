    ```typescript
    export type ComponentStatus = 'healthy' | 'degraded' | 'down';

    export interface ComponentHealth {
      status: ComponentStatus;
      detail?: string;
    }

    export type HealthProvider = () => ComponentHealth;

    export interface HealthCheckResult {
      status: ComponentStatus;
      components: Record<string, ComponentHealth>;
      uptime: number;
      version: string;
      timestamp: number;
    }
    ```

    **Class HealthService:**
    - Constructor takes `botEventBus` instance, `alertStore: AlertStore`, and `config: { alertCooldownMs: number }`. Read version from package.json using `createRequire(import.meta.url)('./../../package.json')` -- synchronous JSON read.
    - `private providers = new Map<string, HealthProvider>()`
    - `private previousStatus = new Map<string, ComponentStatus>()` -- tracks last known status per component for transition detection
    - `private cooldowns = new Map<string, number>()` -- maps composite key `${alertType}:${source}` to last-emitted timestamp
    - `register(name: string, provider: HealthProvider): void` -- stores provider in Map
    - `check(): HealthCheckResult` -- iterates all providers, computes each ComponentHealth, determines aggregate via worst-of logic, calls `this.detectTransitions(components)` to handle alert emission, returns result object with `process.uptime()` for uptime.
    - `private detectTransitions(components: Record<string, ComponentHealth>): void` -- for each component, compares current status to previousStatus. If status worsened (healthy->degraded, healthy->down, degraded->down): emit alert with `severity` based on new status ('warn' for degraded, 'error' for down), `alertSource` mapped from component name (detection->'detection', rpc->'rpc', safety->'api', execution->'api'). If status improved (any->better): emit recovery alert with severity='info'. Update previousStatus. All emissions go through `emitAlert()`.
    - `private emitAlert(type: string, severity: 'info' | 'warn' | 'error', source: string, message: string): void` -- checks cooldown. Composite key = `${type}:${source}`. If `Date.now() - lastEmit < alertCooldownMs`, skip. Otherwise: emit SYSTEM_ALERT via botEventBus (`{ type: 'SYSTEM_ALERT', mint: '', ts: Date.now(), detail: message, severity, alertSource: source as any }`), persist to alertStore (`{ timestamp: Date.now(), type, severity, source, message }`), update cooldown Map.
    - `private resetCooldown(source: string): void` -- removes all cooldown entries whose key ends with `:${source}`. Called on recovery events per D-11.

    Use `createModuleLogger('health-service')` for internal logging.

  <action>
    Create `src/monitoring/metrics-tracker.ts`:

    **Types to export:**
    ```typescript
    export interface EndpointStats {
      p50: number;
      p99: number;
      errorRate: number;
      count: number;
    }
    ```

    **Class MetricsTracker:**
    - Constructor takes `windowMs: number = 300_000` (5 minutes per D-18). Starts a periodic prune timer every 60 seconds via `setInterval` to prevent memory growth (Pitfall 5). Store timer ref for cleanup.
    - `private windows = new Map<string, Array<{ latencyMs: number; success: boolean; ts: number }>>()`
    - `record(endpoint: string, latencyMs: number, success: boolean): void` -- pushes `{ latencyMs, success, ts: Date.now() }` to the endpoint's array. Creates array if not present.
    - `getStats(endpoint: string): EndpointStats` -- computes cutoff = `Date.now() - this.windowMs`. Filters entries for this endpoint to those with `ts >= cutoff`. Replaces the array in the Map with the filtered entries (in-place prune). If empty: return `{ p50: 0, p99: 0, errorRate: 0, count: 0 }`. Otherwise: sort latencies ascending, compute p50 = `latencies[Math.floor(latencies.length * 0.5)]`, p99 = `latencies[Math.floor(latencies.length * 0.99)]`, errorRate = `errors / total`, count = total.
    - `getAllStats(): Record<string, EndpointStats>` -- iterates all keys in windows Map, calls getStats() for each, returns object. Omits endpoints where count=0 after prune.
    - `close(): void` -- clears the prune interval timer. Called on shutdown.
    - `private pruneAll(): void` -- iterates all endpoints, removes entries older than windowMs. Deletes endpoint key entirely if array is empty after prune.

    Use `createModuleLogger('metrics-tracker')` for debug logging only (no info-level noise from recording).
