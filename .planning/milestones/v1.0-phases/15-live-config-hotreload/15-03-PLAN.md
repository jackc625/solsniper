<objective>
Fix SSE EventSource connection being destroyed on tab navigation, which prevents CONFIG_CHANGED (and all other) feed events from reaching the dashboard when the user is not on the Feed tab.

Purpose: The root cause is `key={view}` on `<main>` in app.tsx forcing full unmount of LiveFeed (and its EventSource) on every tab switch. Moving `connectFeed()` to the App component ensures the SSE connection persists for the entire dashboard session, so events emitted while the user is on Settings or Performance tabs are captured in the feedEvents signal and visible when the user returns to Feed.

Output: Two modified files -- app.tsx with App-level SSE lifecycle, LiveFeed.tsx with SSE lifecycle removed.
</objective>
