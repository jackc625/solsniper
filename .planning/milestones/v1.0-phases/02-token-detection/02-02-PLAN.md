**src/detection/pump-portal-listener.ts (DET-01):**

Extend `ResilientWebSocket` from `../core/resilient-ws.js`. Concrete class `PumpPortalListener` that:

1. **Constructor:** Takes `ResilientWsConfig` (url: `wss://pumpportal.fun/api/data`, name: `pump-portal`, plus backoff/heartbeat config from trading config). Also accepts an `EventEmitter<DetectorEvents>` or callback for emitting TokenEvent.

2. **getSubscriptions():** Returns `[{ method: 'subscribeNewToken' }]`. Per research, do NOT also subscribe to `subscribeMigration` — the migration event produces a duplicate mint that the dedup layer would drop anyway. Keep it simple. One connection, one subscription per research anti-pattern guidance (never open multiple connections to PumpPortal).

3. **onMessage(data: string):**
   - Set `const detectedAt = Date.now()` as the FIRST line (per research Pitfall 4 — latency measurement must be before parsing).
   - `JSON.parse(data)` in try/catch. On first message ever received, log the raw event at debug level to validate PumpPortal field presence (research Open Question 2).
   - If `raw.txType === 'create'`: construct `TokenEvent` with `mint: raw.mint`, `source: 'pumpportal'`, `detectedAt`, and optional fields: `name: raw.name`, `symbol: raw.symbol`, `uri: raw.uri`, `creator: raw.traderPublicKey`, `bondingCurveKey: raw.bondingCurveKey`, `initialBuyAmount: raw.initialBuy`, `marketCapSol: raw.marketCapSol`, `vSolInBondingCurve: raw.vSolInBondingCurve`, `vTokensInBondingCurve: raw.vTokensInBondingCurve`, `signature: raw.signature`. Use optional chaining for medium-confidence fields (name, symbol, uri).
   - Call the provided callback/emit with the TokenEvent.
   - Ignore non-create messages silently.

**src/detection/raydium-listener.ts (DET-02):**

Class `RaydiumListener` that wraps Solana `connection.onLogs()`. NOT extending ResilientWebSocket (onLogs uses the @solana/web3.js internal WebSocket, not a raw ws connection). Instead, implements its own health-check for silent subscription death (research Pitfall 2).

1. **Constructor:** Takes `Connection` (from RpcManager), callback for emitting TokenEvent, and config (silence threshold, etc.).

2. **start():** Creates TWO onLogs subscriptions:
   - **Raydium V4** (`675kPX9MHTjS2zt1qfr1NYHuzeLXfQM9H24wFSUt1Mp8`): filter for `'initialize2'` in logs. On match, call `handleRaydiumPool(signature)`.
   - **PumpSwap** (`pAMMBay6oceH9fJKBRHGP5D4bD4sWpmSwMn52FMfXEA`): filter for `'CreatePool'` or `'Instruction: CreatePool'` in logs. On match, call `handlePumpSwapPool(signature)`. Log a debug message on first detection to validate the filter string (research Open Question 1).
   - Both use `'processed'` commitment for lowest latency (per research anti-pattern: never use 'finalized').
   - Store subscription IDs for cleanup.

3. **handleRaydiumPool(signature):**
   - `const detectedAt = Date.now()` first.
   - `connection.getParsedTransaction(signature, { maxSupportedTransactionVersion: 0, commitment: 'confirmed' })`.
   - Find the Raydium V4 instruction, extract accounts at indices 8 and 9 for token mints. Identify non-SOL mint (SOL = `So11111111111111111111111111111111111111112`).
   - Emit `TokenEvent` with `source: 'raydium'`.
   - Wrap in try/catch — log error and continue on parse failure.

4. **handlePumpSwapPool(signature):**
   - Similar to Raydium handler but for PumpSwap. Parse transaction to extract token mint.
   - Emit `TokenEvent` with `source: 'pumpswap'`.
   - NOTE: PumpSwap account indices may differ from Raydium. Log the full instruction accounts at debug level on first detection to identify correct indices. Use defensive coding — if accounts array is too short, log warning and skip.

5. **Health check (research Pitfall 2):** Track `lastEventAt = Date.now()` updated on every log callback (any event, not just matching ones). Start a 60-second `setInterval` that checks `Date.now() - lastEventAt > SILENCE_THRESHOLD_MS` (default: 120000 = 2 minutes). If stale, log warning, remove old subscriptions, and recreate them. This handles silent onLogs death where @solana/web3.js reconnects internally but doesn't replay subscriptions.

6. **stop():** Remove all onLogs subscriptions, clear health check timer.

**src/detection/detection-manager.ts:**

Class `DetectionManager` extends `EventEmitter<DetectorEvents>` that orchestrates both listeners.

1. **Constructor:** Takes `env` config (PUMPPORTAL_ENABLED, RAYDIUM_ENABLED), `tradingConfig.detection` for WS config, `Connection` from RpcManager. Creates PumpPortalListener and RaydiumListener instances based on enabled flags.

2. **start():**
   - If PUMPPORTAL_ENABLED: create PumpPortalListener with config `{ url: 'wss://pumpportal.fun/api/data', name: 'pump-portal', baseBackoffMs, maxBackoffMs, heartbeatIntervalMs, excessiveReconnectThreshold, excessiveReconnectWindowMs }` from trading config. Pass `this.handleTokenEvent.bind(this)` as callback. Call `.connect()`.
   - If RAYDIUM_ENABLED: create RaydiumListener with connection and callback. Call `.start()`.
   - Start stats timer: `setInterval(() => this.logStats(), tradingConfig.detection.statsIntervalMs)` (default 15 minutes per user spec).
   - Log which sources are enabled at info level.

3. **handleTokenEvent(event: TokenEvent):**
   - **Dedup:** Check `seenMints` Map (not Set — use `Map<string, number>` with timestamp for eviction). If mint already seen, log at debug level ("Duplicate token event skipped") and return.
   - Add to seenMints with current timestamp.
   - Increment stats counters.
   - **Pre-filter:** Call `preFilter(event)`. Calculate latency: `Date.now() - event.detectedAt`.
   - **One-liner log (per user spec):** `log.info({ mint: event.mint, source: event.source, latencyMs, preFilter: filterResult }, 'Token detected')` — compact, one line per event.
   - If pre-filter fails: increment filtered counter, log at debug level with reason (per user spec: "Filtered-out tokens logged at debug level"), return.
   - Emit `'token'` event with the TokenEvent for downstream consumers (safety pipeline in Phase 3).

4. **logStats():**
   - Log at info level: `{ total: stats.detected, filtered: stats.filtered, bySource: stats.bySource, dedupDropped: stats.dedupDropped }` — "Detection stats (15m)" message.
   - **Dedup eviction (research Pitfall 5):** During stats logging, prune `seenMints` entries older than `tradingConfig.detection.dedupWindowMs` (default 1 hour). This prevents unbounded memory growth (pump.fun creates thousands of tokens daily).

5. **stop():** Stop PumpPortalListener (`.close()`), stop RaydiumListener (`.stop()`), clear stats timer.
