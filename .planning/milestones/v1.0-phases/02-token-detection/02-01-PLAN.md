**src/core/resilient-ws.ts:** Create an abstract class that wraps the `ws` library WebSocket with:

1. **Connection lifecycle:** `connect()` creates WebSocket to `config.url`. On `open`: reset backoff, start heartbeat, call `replaySubscriptions()`. On `message`: reset heartbeat timer, call abstract `onMessage(data: string)`. On `close`/`error`: stop heartbeat, schedule reconnect (unless `close()` was called explicitly).

2. **Exponential backoff reconnection (DET-03):** Start at `config.baseBackoffMs` (user spec: 2-5s), multiply by 2 on each attempt, cap at `config.maxBackoffMs` (user spec: 60s). Add 10-15% random jitter per user/research spec. Reset backoff to base on successful connection.

3. **Heartbeat (DET-04):** `setInterval` sends `ws.ping()` every `config.heartbeatIntervalMs` (default 30s). Track `lastMessageAt` timestamp, reset on every `message` event. If no message received within `heartbeatIntervalMs * 2`, call `ws.terminate()` to force close (which triggers reconnect via close handler). This detects silent connection death.

4. **Reconnection alerting (DET-05):** Maintain `reconnectTimestamps: number[]` array. On each reconnect, push `Date.now()`, prune entries older than `config.excessiveReconnectWindowMs`. If array length >= `config.excessiveReconnectThreshold`, log warning via pino. Log every reconnection attempt at info level with reconnectCount and backoffMs.

5. **Abstract methods:** `protected abstract onMessage(data: string): void` and `protected abstract getSubscriptions(): object[]` (returns JSON payloads to send on open).

6. **Cleanup:** `close()` sets `closed = true`, stops heartbeat, clears reconnect timer, closes WebSocket. `getReconnectCount()` returns total reconnect count.

Import `WebSocket` from 'ws' (default import). Import `createModuleLogger` from './logger.js'. Use `{ EventEmitter }` from 'eventemitter3' if events needed, but prefer keeping this as a simple class with abstract methods rather than adding event complexity.

IMPORTANT: Use `import WebSocket from 'ws'` — ws exports default in ESM. PumpPortal server supports standard WebSocket ping/pong per research.
