# Phase 20 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Classification: Backend-Only

Phase 20 introduces **no frontend changes**. Per CONTEXT.md: "No new trading features or dashboard UI -- strictly infrastructure for Phase 21 to consume."

This phase produces three new API endpoints (GET /api/health, GET /api/metrics, GET /api/alerts) and one new SSE event type (SYSTEM_ALERT). All visual consumption of these APIs happens in Phase 21 (Dashboard Overhaul), which carries the `UI hint: yes` marker.

The remainder of this document defines the **API response shape contract** that Phase 21's UI will consume. These are the data contracts that downstream UI work depends on -- not visual specifications.

## Copywriting Contract

Phase 20 has no user-facing UI copy. The following defines the **API response field semantics** that Phase 21 will render.

### Health Status Labels (REL-01 -- consumed by DASH-10)

| Status Value | Meaning | Phase 21 Display Intent |
|--------------|---------|------------------------|
| `"healthy"` | Component operating normally | Green indicator |
| `"degraded"` | Component functional but impaired (e.g., no activity in window, backup RPC active) | Amber/yellow indicator |
| `"down"` | Component non-functional (disconnect, crash) | Red indicator |

### Alert Severity Labels (REL-02 -- consumed by DASH-10)

| Severity Value | Meaning |
|----------------|---------|
| `"info"` | Recovery notification or informational state change |
| `"warn"` | Degraded condition requiring attention |
| `"error"` | Critical failure requiring immediate attention |

### Alert Source Labels (REL-02 -- consumed by DASH-10)

| Source Value | Human-Readable Label |
|--------------|---------------------|
| `"detection"` | Detection Feeds |
| `"rpc"` | RPC Connections |
| `"api"` | External APIs |
| `"rateLimit"` | Rate Limiting |

### API Response Shapes (data contracts for Phase 21)

**GET /api/health:**
```json
{
  "status": "healthy | degraded | down",
  "components": {
    "detection": { "status": "healthy | degraded | down", "detail": "string" },
    "rpc": { "status": "healthy | degraded | down", "detail": "string" },
    "safety": { "status": "healthy | degraded | down", "detail": "string" },
    "execution": { "status": "healthy | degraded | down", "detail": "string" }
  },
  "uptime": 12345,
  "version": "1.1.0",
  "timestamp": 1711756800000
}
```

**GET /api/metrics:**
```json
{
  "endpoints": {
    "helius:fee-estimate": { "p50": 45, "p99": 210, "errorRate": 0.02, "count": 156 },
    "jupiter:quote": { "p50": 120, "p99": 580, "errorRate": 0.01, "count": 89 }
  },
  "windowMs": 300000
}
```

**GET /api/alerts?page=1&limit=50:**
```json
{
  "alerts": [
    {
      "id": 1,
      "timestamp": 1711756800000,
      "type": "disconnect",
      "severity": "error",
      "source": "detection",
      "message": "PumpPortal WebSocket disconnected"
    }
  ],
  "total": 42,
  "page": 1,
  "limit": 50
}
```

**SYSTEM_ALERT SSE event (via BotEventBus):**
```json
{
  "type": "SYSTEM_ALERT",
  "mint": "",
  "ts": 1711756800000,
  "detail": "PumpPortal WebSocket disconnected",
  "severity": "error",
  "alertSource": "detection"
}
```
