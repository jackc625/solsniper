<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete web dashboard with:
- Fastify HTTP server running in-process on port 3001 (or DASHBOARD_PORT)
- SSE stream at /events fed by BotEventBus (TOKEN_DETECTED, BUY_SENT, BUY_CONFIRMED, BUY_FAILED, SELL_TRIGGERED, SELL_CONFIRMED, SELL_FAILED, ERROR)
- GET /api/trades — active MONITORING positions with entry P&L
- GET /api/stats — portfolio summary (P&L, win rate, positions)
- GET/POST /api/config — runtime config read and mutation
- Preact SPA: Header bar, Live Feed tab, Performance tab, Settings tab
- Dark terminal aesthetic, monospace font, green/red P&L coloring
- Feed badge colors: DETECTED=gray, BUY=blue, CONFIRMED=green, SELL=yellow, ERROR/FAILED=red
- Auto-scroll with "Resume Live" button when scrolled up
- Edit-then-Save settings form with success/error feedback
  </what-built>
  <how-to-verify>
1. Start the bot (ensure a valid .env exists with real or test RPC URLs):
   ```
   pnpm dev
   ```
   Expected: Log line "Dashboard HTTP server listening" with port 3001

2. Open browser at http://localhost:3001
   Expected: Dashboard loads — dark background, "◎ SOLSNIPER" header, three tabs visible

3. Check Live Feed tab:
   - Expected: "Waiting for events..." if no trading activity, OR recent events if bot detected tokens
   - If bot is running with PUMPPORTAL_ENABLED=true, trigger any token detection — event should appear within seconds
   - Manually scroll up in the feed — "Resume Live" button should appear; clicking it re-enables auto-scroll

4. Check Performance tab:
   - Expected: Table with active positions OR "No active positions." message
   - Click Refresh button — data reloads

5. Check Settings tab:
   - Expected: Form fields populated with current config values (buyAmountSol, minSafetyScore, etc.)
   - Change one value (e.g. minSafetyScore from current to current+1)
   - Click Save — expected: "Saved." appears briefly in green
   - Confirm: GET http://localhost:3001/api/config returns the updated value

6. Check Header bar:
   - Expected: P&L shown (0.0000 SOL if no trades), Win Rate shown, Open Positions shown
   - Values should update every ~5 seconds

7. Optional — API key auth (if DASHBOARD_API_KEY is set in .env):
   - curl -X GET http://localhost:3001/api/config → expected 401 Unauthorized
   - curl -X GET http://localhost:3001/api/config -H "x-dashboard-key: YOUR_KEY" → expected 200

8. Graceful shutdown:
   - Press Ctrl+C in the bot terminal
   - Expected: "Shutdown complete" log; browser dashboard shows connection lost (EventSource will auto-retry)
  </how-to-verify>
