    **SafetyPipeline class (`safety-pipeline.ts`)**

    Export: `class SafetyPipeline`

    Constructor takes:
    - `connection: Connection` (for Tier 1 authority and Tier 2 holder checks)
    - `tradingConfig: TradingConfig` (for minSafetyScore, safety weights, thresholds, timeouts)
    - `env: Env` (for RUGCHECK_API_KEY, HELIUS_API_KEY)

    Initialize in constructor:
    - `SafetyCache` instance with `tradingConfig.safety.cacheTtlMs`
    - `Blocklist` instance with `tradingConfig.safety.blocklistPath`, call `load()` immediately
    - `createModuleLogger('safety-pipeline')` for logging

    **Main method: `async evaluate(event: TokenEvent): Promise<SafetyResult>`**

    Implementation per research Pattern 1:

    ```
    1. Cache check: if SafetyCache has a result for event.mint, return it immediately with a debug log
    2. Record startTime = Date.now()
    3. TIER 1 (hard blocks, parallel, SAF-04):
       - Run Promise.all([checkAuthorities(event.mint, connection), checkSellRoute(event.mint)])
       - checkAuthorities returns [mintAuthResult, freezeAuthResult] -- destructure into two CheckResults
       - Collect all three Tier 1 results: [mintAuthResult, freezeAuthResult, sellRouteResult]
       - If ANY Tier 1 check has pass=false, build and return REJECTED SafetyResult immediately
         - aggregateScore: 0
         - rejectionReasons: list which Tier 1 checks failed with their details
         - Log REJECTION at info level with mint, source, rejectionReasons, durationMs (per user decision: "detailed rejection logging")
         - Cache the rejection result
         - return early

    4. TIER 2 + TIER 3 (scoring signals, parallel with timeouts, research Pitfall 5):
       - Create AbortSignal.timeout() for tier2 and tier3 separately
       - Run Promise.allSettled([
           checkRugCheck(event.mint, env.RUGCHECK_API_KEY, tier2Signal),
           checkHolderConcentration(event.mint, connection, tradingConfig.safety.holder),
           checkCreatorHistory(event.creator, env.HELIUS_API_KEY, blocklist, tier3Signal),
         ])
       - For each settled result:
         - fulfilled: use the CheckResult value
         - rejected: create pessimistic CheckResult { pass: true, score: 0, source: '...', detail: 'timeout_or_error' }

    5. SOFT BLOCK CHECK (per user decision: "individual Tier 2/3 checks CAN independently reject"):
       - If holderConcentration result has pass=false (soft block triggered by thresholds), add to rejectionReasons
       - If creatorHistory result has pass=false (blocklisted or 10+ mints), add to rejectionReasons
       - If any soft block triggered, build REJECTED SafetyResult, cache it, log detailed rejection, return

    6. AGGREGATE SCORE (research Pattern 10):
       - Extract scores from each Tier 2/3 check (default to 0 if undefined -- pessimistic)
       - Compute: aggregateScore = Math.round(
           (rugCheckScore / 100) * weights.rugCheck +
           (holderScore / 100) * weights.holder +
           (creatorScore / 100) * weights.creator
         )
       - If aggregateScore < tradingConfig.minSafetyScore:
           Build REJECTED SafetyResult with rejectionReasons: ["aggregate_score={N} below threshold={minSafetyScore}"]
           Log detailed rejection: { mint, source, aggregateScore, minSafetyScore, tier2Scores, tier3Scores, durationMs }
           Cache and return

    7. PASSED:
       - Build PASSED SafetyResult: pass=true, aggregateScore, all tier results
       - Log at info level: { mint, source, aggregateScore, durationMs } "Token passed safety pipeline"
       - Cache and return
    ```

    **Helper: `buildSafetyResult(pass, mint, score, tier1, tier2, tier3, rejectionReasons, startTime): SafetyResult`**

    **Detailed rejection log format (per user decision):**
    ```typescript
    log.info({
      mint: event.mint,
      source: event.source,
      decision: pass ? 'PASSED' : 'REJECTED',
      aggregateScore,
      minSafetyScore: tradingConfig.minSafetyScore,
      rejectionReasons,
      tier1: tier1Results.map(r => ({ source: r.source, pass: r.pass, detail: r.detail })),
      tier2: tier2Results.map(r => ({ source: r.source, pass: r.pass, score: r.score, detail: r.detail })),
      tier3: tier3Results.map(r => ({ source: r.source, pass: r.pass, score: r.score, detail: r.detail })),
      durationMs: Date.now() - startTime,
    }, pass ? 'Token passed safety pipeline' : 'Token rejected by safety pipeline');
    ```

      4. Rejects with aggregateScore=0 when no sell route (Tier 1 hard block)
      5. Rejects via soft block when holder concentration exceeds threshold (even if aggregate would pass)
      6. Rejects via soft block when creator is blocklisted
      7. Passes with aggregate score above minSafetyScore threshold
      8. Rejects with aggregate score below minSafetyScore threshold

    - AbortSignal.timeout() is Node 18+ built-in -- no library needed
