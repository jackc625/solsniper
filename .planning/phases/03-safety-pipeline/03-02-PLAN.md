<objective>
Implement Tier 2 (RugCheck API, holder concentration) and Tier 3 (creator wallet history) async safety checks. These are scoring signals with configurable soft-block thresholds, not hard blocks. All three handle API failures pessimistically (timeout/error = worst-case score contribution of 0). The creator check uses the Blocklist from Plan 01 for instant rejection of known-bad creators.

Purpose: Tier 2/3 checks provide the scoring signals that feed into the aggregate safety score. Without them, the pipeline can only hard-block on authority/route checks but cannot score the relative risk of passing tokens. Holder concentration and creator history are the strongest rug indicators beyond authority checks.

Output: Three tested check functions ready for the SafetyPipeline orchestrator in Plan 03.
</objective>

    Export: `checkHolderConcentration(mint: string, connection: Connection, config: { top1SoftBlockThreshold: number; top10SoftBlockThreshold: number }): Promise<CheckResult>`

    Implementation per research Pattern 5:
    - Call `connection.getTokenLargestAccounts(new PublicKey(mint))` to get 20 largest token accounts
    - Call `connection.getTokenSupply(new PublicKey(mint))` to get total supply
    - For each of the 20 largest accounts, call `connection.getParsedAccountInfo(account.address)` to resolve the owner wallet address (research Pitfall 1: token accounts != wallets)
    - Define `SYSTEM_ACCOUNTS` Set containing known system addresses (per research code examples):
      - `11111111111111111111111111111111` (System Program)
      - `TokenkegQfeZyiNwAJbNbGKPFXCWuBvf9Ss623VQ5DA` (Token Program)
      - `TokenzQdBNbLqP5VEhdkAS6EPFLC1PHnBqCXEpPxuEb` (Token-2022)
      - `1nc1nerator11111111111111111111111111111111` (Incinerator)
    - Also exclude accounts whose owner is a known program (not a user wallet): check if `ownerAddress` matches any known program ID including `6EF8rrecthR5Dkzon8Nwu78hRvfCKubJ14M5uBEwF6P` (Pump.fun program -- for PDA bonding curve exclusion per research open question 3)
    - Filter to only non-system owner wallets, sort by amount descending
    - Compute `top1Pct = ownerAmounts[0].amount / totalSupply` and `top10Pct = sum(top10) / totalSupply`
    - **Soft block logic (per user decision):**
      - `top1Pct > config.top1SoftBlockThreshold` (default 0.25) -> `pass: false`, detail shows actual vs threshold
      - `top10Pct > config.top10SoftBlockThreshold` (default 0.50) -> `pass: false`, detail shows actual vs threshold
    - Score: `Math.max(0, 100 - Math.round(top10Pct * 100))` -- lower concentration = higher safety score
    - `source: 'holder_concentration'`
    - On error: pessimistic -- `pass: false`, `score: 0`, `detail: 'error: ...'`
    - Edge case: if totalSupply is 0, return `pass: false`, `score: 0`, `detail: 'zero total supply'`
