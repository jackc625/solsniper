    **src/core/rpc-manager.ts (OPS-03):**
    - Import { Connection, Commitment } from '@solana/web3.js'
    - Import EventEmitter from 'eventemitter3'
    - Import { RpcManagerEvents } from '../types/index.js'
    - Import { createModuleLogger } from './logger.js'
    - Create module logger: `const log = createModuleLogger('rpc-manager')`

    Class `RpcManager extends EventEmitter<RpcManagerEvents>`:
    - Private fields: primary (Connection), backup (Connection), state ('primary' | 'backup'), consecutiveFailures (number, init 0), recoveryTimer (NodeJS.Timeout | null), FAILURE_THRESHOLD (3), RECOVERY_INTERVAL_MS (10_000)
    - Constructor(primaryUrl: string, backupUrl: string, commitment: Commitment = 'confirmed'):
      - Create both Connection instances with { commitment }
      - Log startup: `log.info({ primaryUrl: primaryUrl.replace(/api-key=.*/, 'api-key=***'), commitment }, 'RPC manager initialized')`
      - CRITICAL: Mask API keys in RPC URLs before logging (they often contain keys in query params)

    - `getConnection(): Connection` — returns primary or backup based on state
    - `getState(): string` — returns current state (for logging/monitoring)

    - `recordSuccess(): void` — resets consecutiveFailures to 0

    - `recordFailure(reason: string): void`:
      - Increment consecutiveFailures
      - Emit 'degraded' with { endpoint: current state, consecutiveFailures }
      - Log at warn level: { consecutiveFailures, threshold: FAILURE_THRESHOLD }, 'RPC failure recorded'
      - If state === 'primary' AND consecutiveFailures >= FAILURE_THRESHOLD: call switchToBackup(reason)

    - `private switchToBackup(reason: string): void`:
      - Set state = 'backup', reset consecutiveFailures = 0
      - Emit 'failover' with { from: 'primary', to: 'backup', reason, consecutiveFailures: FAILURE_THRESHOLD }
      - Log at warn level: { reason }, 'Switched to backup RPC'
      - Call startRecoveryPolling()

    - `private startRecoveryPolling(): void`:
      - Set interval every RECOVERY_INTERVAL_MS
      - In interval callback: try primary.getSlot(), if success: set state = 'primary', reset failures, stop polling, emit 'recovered', log info
      - On catch: do nothing (primary still down)

    - `private stopRecoveryPolling(): void`:
      - clearInterval if timer exists, set to null

    - `close(): void`:
      - Call stopRecoveryPolling()
      - Log debug: 'RPC manager closed'

    **Shutdown handler function:**
    ```typescript
    let isShuttingDown = false;

    async function shutdown(signal: string, rpcManager: RpcManager): Promise<void> {
      if (isShuttingDown) return;  // Prevent double-shutdown
      isShuttingDown = true;

      log.info({ signal }, 'Shutdown signal received');

      const timeout = setTimeout(() => {
        log.warn('Graceful shutdown timed out after 5s — forcing exit');
        process.exit(1);
      }, 5000);
      timeout.unref();

      try {
        // 1. Close RPC health check timers
        rpcManager.close();

        // 2. Phase 2+: close WebSocket listeners
        // 3. Phase 4+: flush SQLite writes

        // 4. Flush pino logger (ensures buffered logs written)
        log.info('Shutdown complete');
        await new Promise<void>((resolve) => logger.flush(resolve));
      } finally {
        clearTimeout(timeout);
        process.exit(0);
      }
    }
    ```

    **Main startup function:**
    ```typescript
    async function main(): Promise<void> {
      log.info({ nodeEnv: env.NODE_ENV, logLevel: env.LOG_LEVEL }, 'SolSniper starting');

      // 1. Load wallet (validates private key)
      const publicKey = getWalletPublicKey();
      log.info({ publicKey }, 'Wallet loaded');

      // 2. Initialize RPC manager
      const rpcManager = new RpcManager(
        env.SOLSNIPER_RPC_URL,
        env.SOLSNIPER_RPC_BACKUP_URL
      );

      // 3. Log RPC manager events
      rpcManager.on('failover', (data) => log.warn(data, 'RPC failover triggered'));
      rpcManager.on('recovered', (data) => log.info(data, 'RPC primary recovered'));
      rpcManager.on('degraded', (data) => log.debug(data, 'RPC degraded'));

      // 4. Log trading config (safe — no secrets)
      log.info({ tradingConfig }, 'Trading configuration loaded');

      // 5. Register shutdown handlers
      const handler = (signal: string) => { shutdown(signal, rpcManager); };
      process.on('SIGTERM', () => handler('SIGTERM'));
      process.on('SIGINT', () => handler('SIGINT'));

      log.info('SolSniper ready — waiting for Phase 2 (detection) to be implemented');
      // Phase 2+: start detection listeners, position monitor, etc.
    }

    main().catch((err) => {
      log.fatal({ err }, 'Fatal startup error');
      process.exit(1);
    });
    ```

    CRITICAL initialization order: env.ts (dotenv + zod) -> trading.ts (config.json + zod) -> logger.ts (needs env for level/NODE_ENV) -> wallet.ts (needs env for private key) -> rpc-manager (needs env for URLs) -> shutdown handler (needs rpc-manager reference).

    Note: The import of env.ts happens as a side-effect at module load time. The logger module imports env.ts, so by the time main() runs, config is already validated. If config validation fails, process.exit(1) happens before main() is called.
