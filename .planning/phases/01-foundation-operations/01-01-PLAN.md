    **src/types/index.ts:**
    Create shared types file. For now, export placeholder comment. Will be populated by Plan 02 with RpcManagerEvents etc.

    **src/config/env.ts (OPS-06):**
    - First line: `import 'dotenv/config';` (side-effect import loads .env before anything else)
    - Import { z } from 'zod'
    - Define `EnvSchema = z.object({...})` with:
      - `SOLSNIPER_RPC_URL`: z.string().url()
      - `SOLSNIPER_RPC_BACKUP_URL`: z.string().url()
      - `SOLSNIPER_PRIVATE_KEY`: z.string().min(32) — never echo value in error
      - `NODE_ENV`: z.enum(['development', 'production']).default('development')
      - `LOG_LEVEL`: z.enum(['trace', 'debug', 'info', 'warn', 'error']).default('debug')
    - Use `safeParse` (NOT parse) to collect all errors
    - On failure: console.error each issue as `  [FIELD_NAME] message`, then process.exit(1)
    - Export `env` (typed via z.infer) and `Env` type
    - CRITICAL: The error output for SOLSNIPER_PRIVATE_KEY must NEVER include the actual value. Zod's default error messages are safe (they say "too short" etc.), but verify no custom transforms echo it.

    **src/config/trading.ts (OPS-06):**
    - Import { z } from 'zod', import fs from 'node:fs', import path from 'node:path'
    - Read config.json via `fs.readFileSync(path.resolve('config.json'), 'utf-8')`, then JSON.parse
    - If file not found: console.error with clear message, process.exit(1)
    - Define `TradingConfigSchema = z.object({...})` with:
      - `buyAmountSol`: z.number().positive().max(10)
      - `maxSlippageBps`: z.number().int().min(50).max(4900)
      - `maxConcurrentPositions`: z.number().int().min(1).max(50)
      - `stopLossPct`: z.number().negative()
      - `takeProfitPct`: z.number().positive()
      - `minSafetyScore`: z.number().int().min(0).max(100)
    - Use `safeParse`, list all errors on failure, process.exit(1)
    - Export `tradingConfig` and `TradingConfig` type

    **src/core/logger.ts (OPS-01, OPS-02):**
    - Import pino from 'pino'
    - Import { env } from '../config/env.js'
    - Create base logger:
      - level: env.LOG_LEVEL
      - transport: isDev ? { target: 'pino-pretty', options: { colorize: true, translateTime: 'SYS:standard' } } : undefined
      - serializers: add a custom serializer that strips any object key containing 'PRIVATE_KEY' or 'SECRET' (belt-and-suspenders protection per OPS-04)
    - Export `logger` (base pino instance)
    - Export `createModuleLogger(module: string)` — returns `logger.child({ module })`
    - Export `createTradeLogger(tradeId: string, module?: string)` — returns `logger.child({ tradeId, ...(module ? { module } : {}) })`
    - Export `withLatency<T>(log: pino.Logger, operation: string, fn: () => Promise<T>): Promise<T>`:
      - Record Date.now() before fn()
      - On success: log.debug({ operation, latencyMs }, `${operation} completed`)
      - On error: log.error({ operation, latencyMs, err }, `${operation} failed`), then re-throw
    - Log destination: stdout only (per Claude's Discretion — redirect to file at process level)
    - Default dev level: debug (per Claude's Discretion — trace is too verbose)
    - Latency logging: always log, no threshold gate (per Claude's Discretion — data only useful when complete)
  </action>
