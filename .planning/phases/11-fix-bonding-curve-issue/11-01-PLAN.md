<objective>
Fix the holder concentration check that incorrectly counts the Pump.fun bonding curve PDA as a whale holder. The bonding curve is a per-token derived address (not the static Pump.fun program ID), so it never matched the static SYSTEM_ACCOUNTS exclusion set. This caused ~90% of new Pump.fun tokens to fail with 75-90%+ top-1 concentration.

Purpose: Unblock Pump.fun token buying by correctly excluding the bonding curve PDA from holder concentration analysis.
Output: Fixed tier2-holder.ts with PDA exclusion, source-threaded safety-pipeline.ts, updated test suites.
</objective>
