**RecoverySummary interface:**
```typescript
export interface RecoverySummary {
  monitoring: number;          // MONITORING trades loaded as-is
  sellingResumed: number;      // SELLING trades stepped back + SellLadder called
  sellingCompleted: number;    // SELLING trades with empty wallet → marked COMPLETED
  buyingRecovered: number;     // BUYING trades with balance > 0 → transitioned to MONITORING
  buyingUnrecovered: number;   // BUYING trades with balance = 0 or RPC timeout → marked FAILED
  detectedDiscarded: number;   // DETECTED trades → marked FAILED
}
```

**RecoveryManager class:**
```typescript
import { PublicKey, Connection } from '@solana/web3.js';
import { TOKEN_2022_PROGRAM_ID } from '@solana/spl-token';
import type { TradeStore } from '../persistence/trade-store.js';
import type { SellLadder } from '../execution/sell/sell-ladder.js';
import { createModuleLogger } from '../core/logger.js';
import type { Trade } from '../types/index.js';

const log = createModuleLogger('recovery-manager');

const RPC_TIMEOUT_MS = 5000;  // Per-trade timeout — one slow RPC doesn't block others

export class RecoveryManager {
  constructor(
    private readonly tradeStore: TradeStore,
    private readonly connection: Connection,
    private readonly walletPubKey: PublicKey,
    private readonly sellLadder: SellLadder,
  ) {}

  async run(): Promise<RecoverySummary> { ... }

  private async getWalletTokenBalance(mint: string): Promise<bigint> { ... }
}
```

**getWalletTokenBalance(mint) implementation:**
- Parse mint string to PublicKey
- Run two parallel calls:
  1. `connection.getTokenAccountsByOwner(this.walletPubKey, { mint: mintPubKey }, { encoding: 'jsonParsed' })` — legacy SPL
  2. `connection.getParsedTokenAccountsByOwner(this.walletPubKey, { programId: TOKEN_2022_PROGRAM_ID })` — Token-2022, then filter `.value` by `a.account.data.parsed?.info?.mint === mint`
- Sum all `BigInt(account.data.parsed.info.tokenAmount.amount)` across both results
- Return total balance as `bigint`
- Use `(acct.account.data as any).parsed` for type access (jsonParsed response is typed as `ParsedAccountData | Buffer` in web3.js v1)

**run() implementation — follow this order exactly:**

```
Step 1: Discard DETECTED trades
  - call tradeStore.getDetectedTrades()
  - for each: tradeStore.transition(trade.mint, 'DETECTED', 'FAILED', { errorMessage: 'RECOVERY: DETECTED trade discarded' })
  - count as detectedDiscarded

Step 2: Deduplicate SELLING trades by mint
  - call tradeStore.getSellingTrades() — already ORDER BY updated_at DESC so first in group = most recent
  - group by mint using Map<string, Trade[]>
  - for each mint group with > 1 entries:
    - log.error({ mint, count }, 'Multiple SELLING records for mint — keeping most recent')
    - for stale trades (group.slice(1)): tradeStore.transitionById(stale.id, stale.mint, 'SELLING', 'FAILED', { errorMessage: 'RECOVERY: duplicate SELLING record' })
  - keep group[0] for each mint as the "current" SELLING trade to process

Step 3: Process SELLING trades (current one per mint)
  - for each current SELLING trade:
    - wrap balance check in withTimeout(getWalletTokenBalance(trade.mint), RPC_TIMEOUT_MS)
    - if balance > 0n:
      - step back: tradeStore.transition(trade.mint, 'SELLING', 'MONITORING')
      - compute tokenAmount: balance (use on-chain balance, not stored amountTokens — handles PumpPortal null case)
      - fire-and-forget: void sellLadder.sell(trade.mint, balance)
      - sellingResumed++
    - if balance === 0n:
      - tradeStore.transition(trade.mint, 'SELLING', 'COMPLETED', { errorMessage: 'RECOVERY: sell may have landed — wallet empty' })
      - log.info({ mint: trade.mint, tradeId: trade.id }, 'SELLING trade completed — wallet empty, sell assumed landed')
      - sellingCompleted++
    - catch (timeout or RPC error):
      - tradeStore.transition(trade.mint, 'SELLING', 'FAILED', { errorMessage: 'RECOVERY: RPC unavailable' })
      - log.warn({ mint: trade.mint, tradeId: trade.id, err }, 'SELLING trade recovery failed — RPC unavailable')
      - buyingUnrecovered++ — NO, count this separately? No: per context, count with sellingCompleted as "unrecovered". Use sellingCompleted++ for RPC error on SELLING? Re-read context: "fail-safe closed — mark trade FAILED and continue". Use a separate counter or fold into sellingCompleted? Research says RecoverySummary has sellingCompleted. Treat RPC-fail-on-SELLING as sellingCompleted (it's gone) but log.warn with 'RECOVERY: SELLING RPC unavailable'.

Step 4: Process BUYING trades
  - call tradeStore.getBuyingTrades()
  - for each:
    - wrap in withTimeout(getWalletTokenBalance(trade.mint), RPC_TIMEOUT_MS)
    - if balance > 0n:
      - tradeStore.transition(trade.mint, 'BUYING', 'MONITORING')
      - log.info({ mint: trade.mint, tradeId: trade.id }, 'BUYING trade recovered — tokens found in wallet')
      - buyingRecovered++
    - if balance === 0n:
      - tradeStore.transition(trade.mint, 'BUYING', 'FAILED', { errorMessage: 'RECOVERY: balance=0 — buy did not land' })
      - log.warn({ mint: trade.mint, tradeId: trade.id }, 'BUYING trade unrecovered — wallet balance zero')
      - buyingUnrecovered++
    - catch:
      - tradeStore.transition(trade.mint, 'BUYING', 'FAILED', { errorMessage: 'RECOVERY: RPC unavailable' })
      - log.warn({ mint: trade.mint, tradeId: trade.id, err }, 'BUYING trade recovery failed — RPC unavailable')
      - buyingUnrecovered++

Step 5: Load MONITORING trades (no wallet check)
  - call tradeStore.getMonitoringTrades()
  - monitoring = trades.length
  - log.debug({ count: monitoring }, 'MONITORING trades loaded as-is')

Step 6: Return summary
  return { monitoring, sellingResumed, sellingCompleted, buyingRecovered, buyingUnrecovered, detectedDiscarded }
```

**withTimeout helper (private or module-level):**
```typescript
function withTimeout<T>(promise: Promise<T>, ms: number): Promise<T> {
  return Promise.race([
    promise,
    new Promise<never>((_, reject) =>
      setTimeout(() => reject(new Error(`RPC timeout after ${ms}ms`)), ms)
    ),
  ]);
}
```

**Important notes:**
- `void sellLadder.sell(...)` — fire-and-forget, same pattern as `void executionEngine.buy()` in index.ts. SellLadder handles its own error logging internally.
- For Token-2022 balance access, the `getParsedTokenAccountsByOwner` response items have type `ParsedAccountInfo`. Cast `.account.data` as `any` to access `.parsed.info.tokenAmount.amount` (web3.js v1 types are incomplete for jsonParsed encoding).
- SELLING step-back: `tradeStore.transition(trade.mint, 'SELLING', 'MONITORING')` followed by `void sellLadder.sell(trade.mint, balance)`. SellLadder internally transitions MONITORING→SELLING as its first action — this is the "step-back" pattern required by SellLadder's design.
- All steps run sequentially (not parallel) to avoid hammering RPC with concurrent calls.
