<objective>
Extend TradeStore with query methods and transitionById so RecoveryManager (Plan 02) can read non-terminal trades by state and perform id-precise state transitions during crash recovery.

Purpose: RecoveryManager needs to iterate trades by state (BUYING, SELLING, MONITORING, DETECTED) to make per-trade recovery decisions. The existing TradeStore only exposes `isActive()`, `createBuyingRecord()`, and `transition()` (by mint+state). Two gaps exist: (1) no way to read full Trade rows by state, (2) `transition()` uses mint+state WHERE clause which is ambiguous when two SELLING rows exist for the same mint.

Output: Extended trade-store.ts with 4 state query methods + `transitionById`. All new methods covered by tests.
</objective>

**transitionById — id-precise transition for duplicate-SELLING deduplication:**
```typescript
transitionById(
  id: number,
  mint: string,       // required to update activeMints correctly
  from: TradeState,
  to: TradeState,
  extra: Partial<Pick<Trade, 'errorMessage'>> = {}
): number {
  const now = Date.now();
  const result = this.stmtUpdateStateById.run({
    id,
    state: to,
    now,
    expectedState: from,
    error_message: extra.errorMessage ?? null,
  });
  const changes = result.changes;
  if (changes > 0 && TERMINAL_STATES.has(to)) {
    this.activeMints.delete(mint);
    log.debug({ id, mint, from, to }, 'transitionById: reached terminal state, removed from active set');
  } else if (changes > 0) {
    log.debug({ id, mint, from, to }, 'transitionById: non-terminal state update');
  } else {
    log.warn({ id, mint, from, to }, 'transitionById: optimistic lock miss (state or id mismatch)');
  }
  return changes;
}
```
