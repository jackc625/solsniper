Declared values (must be multiples of 4):

| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--sp-1` | 4px | Icon gaps, inline padding, divider margins |
| sm | `--sp-2` | 8px | Compact element spacing, input padding |
| md | `--sp-3` | 12px | Nav item padding, card inner padding, connection bar |
| lg | `--sp-4` | 16px | Default section padding, inter-card gap, field row gaps |
| xl | `--sp-5` | 20px | Brand area top padding, page header padding |
| 2xl | `--sp-6` | 24px | Page-level padding (Settings grid), save bar |
| 3xl | `--sp-8` | 32px | Major section breaks |

Exceptions:
- Emergency stop button: 44px minimum touch target height (accessibility for critical action)
- STOP confirmation text input: full-width minus 24px horizontal padding

**Source:** Pre-existing `--sp-*` tokens in `dashboard/index.html`. Exception for e-stop is new for DASH-09.

## Interaction Contracts

### Force Sell Flow

1. User clicks FORCE SELL button on open position row
2. Button immediately changes to "SELLING..." amber badge with pulse animation
3. POST `/api/trades/:id/force-sell` fires
4. If 409 Conflict (already selling): no-op, badge already shows SELLING
5. If 200 OK: position row updates via next poll cycle (5s)
6. If error: show inline red error text below the row for 5 seconds

### Pause/Resume Detection Flow

1. User clicks PAUSE or RESUME toggle on Controls page
2. POST `/api/controls/detection` with `{paused: true/false}`
3. Toggle state updates immediately (optimistic)
4. Sidebar connection bar updates to PAUSED (yellow) or CONNECTED (green)
5. If error: revert toggle, show red error text for 5 seconds

### Emergency Stop Flow

1. User clicks EMERGENCY STOP (sidebar or controls page)
2. Confirmation dialog appears with dark overlay
3. User must type "STOP" in text input
4. Confirm button enables only when input matches exactly "STOP"
5. On confirm: POST `/api/controls/emergency-stop`
6. Dialog shows "EXECUTING..." state (button disabled, spinner/pulse)
7. On success: dialog closes, sidebar shows PAUSED state, positions show SELLING badges
8. On error: dialog shows red error message, confirm button re-enables

### Source Filter on Equity Curve

1. User clicks source toggle button (ALL, PUMP, RAY, PSWAP)
2. Active button gets amber styling, previously active reverts to inactive
3. Chart re-renders with filtered data (client-side filter on history)
4. Per-source stat cards always show all sources regardless of filter

### Pipeline Card Expansion

1. User clicks anywhere on collapsed pipeline card
2. Card expands to show per-check detail table
3. Max-height transition: 0 to 300px over 150ms ease
4. Click again to collapse (same as FeedCard pattern)

### Alert History Pagination

1. Initial load: GET `/api/alerts?limit=50`
2. User clicks "Load more"
3. GET `/api/alerts?limit=50&offset={current_count}`
4. New alerts appended below existing list
5. "Load more" button hidden when returned count < limit
