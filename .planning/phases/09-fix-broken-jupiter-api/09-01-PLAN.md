<feature>
  <name>JupiterClient — centralized Jupiter API client with auth + rate limiting</name>
  <files>src/execution/jupiter-client.ts, src/execution/jupiter-client.test.ts, src/config/env.ts, .env.example</files>
  <behavior>
    JupiterClient is a class exported as both the class and a module-level singleton (`jupiterClient`).

    **quote(params, signal?):**
    - Calls GET https://api.jup.ag/swap/v1/quote?{params} with x-api-key header
    - On 200: returns parsed JSON
    - On 429: triggers global cooldown (Retry-After header * 1000 if present, else 10000ms), throws Error('Jupiter rate limited (429)')
    - On any other non-2xx: throws Error('Jupiter quote HTTP {status}')
    - If in cooldown: throws Error('Jupiter rate limited — cooldown active') without making request
    - If signal provided: passes signal to fetch() call

    **swap(body):**
    - Calls POST https://api.jup.ag/swap/v1/swap with x-api-key + Content-Type headers
    - On 200: returns parsed JSON as { swapTransaction: string }
    - On 429: triggers global cooldown, throws
    - On any other non-2xx: throws
    - If in cooldown: throws without making request

    **isRateLimited():** returns true if currently in cooldown
    **cooldownRemainingMs():** returns ms remaining in cooldown (0 if not in cooldown)

    Test cases:
    - quote() includes x-api-key header in fetch call
    - quote() returns parsed JSON on 200
    - quote() throws on non-2xx (e.g., 400, 500)
    - quote() triggers cooldown on 429, subsequent calls throw without fetching
    - quote() respects Retry-After header (converts seconds to ms)
    - quote() falls back to 10s cooldown when no Retry-After header
    - quote() passes AbortSignal to fetch when provided
    - swap() includes x-api-key and Content-Type headers
    - swap() returns { swapTransaction } on 200
    - swap() triggers cooldown on 429
    - swap() throws when in cooldown
    - isRateLimited() returns true during cooldown, false outside
    - cooldownRemainingMs() returns positive value during cooldown, 0 outside
    - Cooldown from quote() blocks swap() (global state)
    - Cooldown expires after duration (use vi.advanceTimersByTime)
  </behavior>
  <implementation>
    1. Add `SOLSNIPER_JUPITER_API_KEY: z.string().min(1, 'Jupiter API key required — get one at https://portal.jup.ag')` to EnvSchema in src/config/env.ts.

    2. Add to .env.example:
    ```
    # Jupiter Swap API key — REQUIRED. Free key at https://portal.jup.ag
    SOLSNIPER_JUPITER_API_KEY=your_jupiter_api_key_here
    ```

    3. Create src/execution/jupiter-client.ts:
    - Class `JupiterClient` with private `cooldownUntil: number = 0`
    - Private `headers()` returns `{ 'x-api-key': env.SOLSNIPER_JUPITER_API_KEY }`
    - Private `isCoolingDown()` checks `Date.now() < this.cooldownUntil`
    - Private `triggerCooldown(retryAfterMs?)` sets `cooldownUntil = Date.now() + (retryAfterMs ?? 10_000)` and logs warn
    - `quote(params: URLSearchParams, signal?: AbortSignal): Promise<unknown>` — GET /swap/v1/quote
    - `swap(body: Record<string, unknown>): Promise<{ swapTransaction: string }>` — POST /swap/v1/swap
    - `isRateLimited(): boolean` — public for PositionManager
    - `cooldownRemainingMs(): number` — public for PositionManager
    - Module-level `export const jupiterClient = new JupiterClient()`
    - Also export the class for test isolation: `export { JupiterClient }`

    4. Create src/execution/jupiter-client.test.ts:
    - Use vi.stubGlobal('fetch', mockFetch) pattern (same as tier1-sell-route.test.ts)
    - Use vi.useFakeTimers() for cooldown expiry tests
    - Instantiate fresh `new JupiterClient()` per test for isolation (NOT the singleton)
    - Mock env via vi.mock('../config/env.js') returning `{ env: { SOLSNIPER_JUPITER_API_KEY: 'test-key' } }`
    - NOTE: Since env.ts calls process.exit(1) on validation failure, the test MUST mock the entire env module (not import the real one). Use `vi.mock` to replace the module with a simple object.

    Key anti-patterns to avoid:
    - Do NOT sleep/wait inside quote()/swap() until cooldown expires — throw immediately, let callers handle
    - Do NOT make JupiterClient a function — it needs mutable cooldown state
    - Do NOT use per-file cooldown — global singleton is the requirement
