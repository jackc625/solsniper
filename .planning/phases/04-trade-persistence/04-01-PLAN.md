  <behavior>
    TradeStore must satisfy these exact I/O contracts (write tests for each before implementing):

    **isActive(mint)**
    - Returns false when no trade exists for the mint
    - Returns true after createBuyingRecord(mint) called
    - Returns false after transition(mint, 'BUYING', 'COMPLETED', {}) called
    - Returns false after transition(mint, 'BUYING', 'FAILED', { errorMessage: 'err' }) called
    - Returns false after transition(mint, 'BUYING', 'ABANDONED', {}) called

    **createBuyingRecord(mint)**
    - Inserts a row with state='BUYING' synchronously
    - Throws Error with message containing 'Duplicate buy attempt blocked' if isActive(mint) is already true
    - After throwing, the Set still contains the mint (no partial state)
    - Creates the 'data/' directory if using a file path (constructor responsibility)

    **transition(mint, from, to, extra?)**
    - Returns changes=1 on successful transition
    - Returns changes=0 if current state !== from (optimistic locking)
    - Sets buy_signature when extra.buySignature provided (BUYING → MONITORING)
    - Sets sell_signature when extra.sellSignature provided (MONITORING → SELLING → COMPLETED)
    - Sets error_message when extra.errorMessage provided (any → FAILED)
    - Does NOT remove from active Set for non-terminal transitions (BUYING → MONITORING, MONITORING → SELLING)
    - DOES remove from active Set for terminal transitions (→ COMPLETED, → FAILED, → ABANDONED)

    **startup Set rebuild**
    - When TradeStore is constructed with a DB that has existing non-terminal rows (DETECTED, BUYING, MONITORING, SELLING), isActive() returns true for all those mints
    - When TradeStore is constructed with a DB that has only terminal rows (COMPLETED, FAILED, ABANDONED), isActive() returns false for all those mints

    **close()**
    - Calls db.close() — no error thrown on already-closed DB test needed
  </behavior>
